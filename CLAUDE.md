# CLAUDE.md

## 프로젝트 개요

이 프로젝트는 NestJS 백엔드 프로젝트입니다.

개발자는 NestJS를 처음 사용합니다. 따라서 복잡한 설계보다 NestJS의 기본 구조와 관례를 따르는 것을 우선합니다.

코드는 무조건 쉽고, 명확하고, 읽기 좋아야 합니다.

## 응답 언어

항상 한국어로 응답합니다.

## 핵심 원칙

- 쉬운 코드를 우선합니다.
- 가독성 좋은 코드를 우선합니다.
- NestJS 기본 관례를 따릅니다.
- 과한 설계를 피합니다.
- 필요 없는 추상화를 만들지 않습니다.
- 처음부터 복잡한 폴더 구조를 만들지 않습니다.
- 비즈니스 로직의 흐름이 쉽게 따라가야 합니다.
- 똑똑해 보이는 코드보다 명확한 코드를 선호합니다.
- 처음 NestJS를 배우는 개발자가 이해할 수 있는 구조를 유지합니다.

## 코드 스타일

- 변수명, 함수명, 클래스명, 파일명을 명확하게 작성합니다.
- 의미 없는 축약어를 피합니다.
- 함수는 작고 명확한 책임만 가지게 합니다.
- 중요한 로직을 너무 많은 유틸 함수 뒤에 숨기지 않습니다.
- 조건문과 흐름은 직관적으로 작성합니다.
- 조기 최적화를 하지 않습니다.
- `any` 사용을 피합니다.
- 제네릭을 남용하지 않습니다.
- 타입은 가능한 구체적으로 작성합니다.

## NestJS 기본 구조

NestJS에서는 역할을 명확히 나눕니다.

- `controller`: HTTP 요청과 응답을 담당합니다.
- `service`: 실제 비즈니스 로직을 담당합니다.
- `module`: 관련 기능을 하나로 묶습니다.
- `dto`: 요청 데이터의 형태를 정의합니다.
- `entity` 또는 `model`: 데이터베이스와 연결되는 데이터 구조가 필요할 때 사용합니다.

컨트롤러에는 비즈니스 로직을 넣지 않습니다.

컨트롤러는 주로 다음 역할만 합니다.

- 요청을 받기
- 파라미터나 body를 받기
- 서비스를 호출하기
- 응답을 반환하기

서비스는 다음 역할을 담당합니다.

- 실제 애플리케이션 로직 처리
- 데이터 가공
- 저장소나 외부 API와의 통신이 필요할 경우 처리

## 기본 폴더 구조

처음에는 단순한 구조를 유지합니다.

예시:

```txt
src/
  main.ts
  app.module.ts
  users/
    users.module.ts
    users.controller.ts
    users.service.ts
    dto/
      create-user.dto.ts
      update-user.dto.ts
```
